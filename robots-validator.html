Understanding Robots.txt

The robots.txt file tells search engine crawlers which pages or files they can or can't request from your site. It's one of the foundational elements of technical SEO and proper website management.

Common Directives

User-agent

Specifies which crawler the rules apply to. Use * for all crawlers, or specify names like Googlebot, Bingbot, etc.

User-agent: *

Disallow

Tells crawlers not to access the specified path. Empty value means allow all.

Disallow: /admin/

Allow

Explicitly allows crawling of a path within a disallowed directory. Takes precedence over Disallow.

Allow: /admin/public/

Sitemap

Points crawlers to your XML sitemap for better indexing of your site's pages.

Sitemap: https://example.com/sitemap.xml

Best Practices

  • Always include a Sitemap directive: Helps search engines discover all your pages.
  • Don't block CSS/JS: Googlebot needs to render pages to understand them properly.
  • Block sensitive areas: Admin panels, user data, staging environments.
  • Test before deploying: Use Google Search Console's robots.txt tester.
  • Keep it simple: Complex rules are hard to maintain and debug.
